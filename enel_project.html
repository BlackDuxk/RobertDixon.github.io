Sort and Filter Component

ENEL X | UI/UX Design

Problem

A Global Crisis

Users of Enel’s global applications are not able to quickly find information needed to complete tasks, as a result users requested a tool that would allow them to filter a list of data before viewing it in an application.

Research

Slow And Steady Loses The Race

Research revealed the different filtering parameters users were interested in, as well as how users expected the tool to behave in different applications. Additionally, we found that users didn't necessarily need to filter data before viewing it, but that they felt they did because of Enel's applications' slow load times.

Solutions

Placing A Priority On Preference

In addition to designing a component that allows users to sort and filter data based on requested parameters, we also gave users the ability to save selected parameters to a customizable, named view.

Less Waiting More Browsing

By introducing lazy loading into Enel’s applications, data is loaded based on user interaction rather than all at once when an application initially loads.

Contextualizing The Search Bar

Providing more context to users when they search for data allows them to quickly identify and understand the relationships between different data groups, leading to improved decision-making.

Insight

Ending the game of hide 'n seek

Rounds of usability testing proved that the sort and filter component was flexible enough to handle different use cases present across Enel’s global applications. Additionally, users commented that the component and other solutions were intuitive,  behaved as expected, and made “finding the things we need to find easier”.
